Use this online tool to understand how pure JavaScript authentication works in SPAs and more specifically in AngularJS.

Quick Start

  • Select an authentication provider such as Google, Facebook... in the dropdown list at the top of the screen.
  • Click the buttons to call methods like login(), logout(), etc. for the provider you've selected and inspect the results directly in the page. Note that I have used generic method names to wrap provider-specific code. The inspector will tell you which methods were actually called in the provider's API/SDK to perform login, logout, etc.
  • Grab the code on Github to reuse it in your own applications! More specifically, you might want to look in the auth_providers folder to see how authentication was implemented for a specific provider that you're interested in.

Installation

Follow these instructions to install Auth Inspector locally:

  • Check out the Github repository https://github.com/ngworkshop/auth-inspector.
  • Grab an API Key/Client ID/etc. for the provider(s) you're interested in and copy it in config.js. Visit the provider's page on this site for more specific instructions.
  • Depending on the provider, you might need to whilelist the URLs where your code is going to run. For example, if you're developing on http://localhost and you're using Facebook as your auth provider, you need to create a Facebook App and enter the http://localhost in that app. Again, visit the provider's page on this site for more specific instructions.
    • Grab an API Key/Client ID/etc. for the provider(s) you're interested in and copy it in config.js. Select the provider you're interested in in the dropdown list for more specific instructions.
  • There is no fancy build system or node.js server. The repository contains the entire code required to run the application. You will need a server to serve the static files located in the src folder. If you have a server like Apache installed locally, you could just copy the repository there and browse to the folder. Or you could execute these commands from the prompt: 
    > cd LOCAL_REPO/src
> python -m SimpleHTTPServer 4567

Why this tool?

Because I — Vince Carré — was struggling to wrap my head around authentication in SPAs. There are lots of pieces involved (securing your endpoints, securing the routes of your application...), a lot of techniques (cookies, JSON Web Tokens...) and a lot of authentication providers (google, facebook, auth0...). Hopefully this tool clears up some of the confusion.
